SDIO调试指南

Description of the SDIO device debuging

platform：hi3556av200

SDIO device：rtl8189fs

1.检测SDIO设备

  执行cat /proc/mci/mci_info

  若出现下列信息，则表示SD卡和SDIO设备均被探测到。

MCI0: pluged_connected
        Type: SD card(SDHC)
        Mode: UHS SDR104 
        Speed Class: Class 10
        Uhs Speed Grade: 10MB/sec and above(1h)
        Host work clock: 150MHz
        Card support clock: 150MHz
        Card work clock: 150MHz
        Card error count: 0
MCI1: pluged_connected
        Type: SDIO card Mode: HS SDR104 
        Speed Class: Class 0
        Uhs Speed Grade: Less than 10MB/sec(0h)
        Host work clock: 50MHz
        Card support clock: 50MHz
        Card work clock: 50MHz
        Card error count: 0

2.若SDIO设备未被发现，则按照以下步骤调试

  1. 检索当前设备树
  2. 检查SDIO设备对应mmc总线
  3. 检查SDIO检测功能是否使能
  4. 检查SDIO检测阶段的mmc总线的指令流

  接下来分点介绍各个步骤

3.检索当前设备树

  开发板上路径/sys/firmware/devicetree/base对应了设备树信息

  1. 压缩tar -zcf /app/data/device_tree.tar.gz /sys/firmware/devicetree/base
  2. 在虚拟机上使用解压后使用DTC工具解析：dtc –sort -I fs -O dts sys/firmware/devicetree/base/ > device_tree.out
  3. 比较device_tree.out与dts、dtsi设备树文件，确保SDIO部分一致

4.检查SDIO设备对应mmc总线

  在设备树中，SDIO网卡对应了SDIO1

  对应dtsi文件中mmc2：

mmc2: himci.SD@0x10020000 {
        compatible = "hisilicon,hi3556v200-himci";
        reg = <0x10020000 0x1000>;
        interrupts = <0 31 4>;
        cap-sdio-irq;
        clocks = <&clock HI3556V200_MMC2_CLK>;
        clock-names = "mmc_clk";
        resets = <&clock 0x154 0>;
        reset-names = "mmc_reset";
        max-frequency = <100000000>;
        bus-width = <4>;
        cap-sd-highspeed;
        sd-uhs-sdr12;
        sd-uhs-sdr25;
        sd-uhs-sdr50;
        sd-uhs-sdr104;
        devid = <2>;
        status = "disabled";
};

5.检查SDIO检测功能是否使能

  这个在海思的平台上，由mmc_load.sh中insmod himci.ko等实现，具体可以参考内核中himci目录下代码

6.检查SDIO检测阶段的mmc总线的指令流

  首先，SDIO检测只会在绑定mmc总线的时候进行。开机时会进行总线绑定（bind bus），或者手动bind和unbind bus.

  开机时追踪比较麻烦，所以优先选择手动bind和unbind mmc bus.

  具体操作为：

6.1先修改内核配置，重新编译内核和设备驱动：

Kernel Hacking -> Tracers -> FUNCTION_TRACER
Kernel Hacking -> Tracers -> FUNCTION_GRAPH_TRACER (if possible)
Kernel Hacking -> Tracers -> STACK_TRACER(Trace max stack)
Kernel Hacking -> Tracers -> DYNAMIC_FTRACE(enable/disable ftrace tracepoints dynamically)

6.2使能/sys/kernel/debug下的打印信息

mount -t debugfs none /sys/kernel/debug

6.3使能mmc请求追踪：

echo 1 > /sys/kernel/debug/tracing/events/mmc/mmc_request_done/enable
echo 'cmd_opcode != 53' > /sys/kernel/debug/tracing/events/mmc/mmc_request_done/filter

6.4绑定mmc bus：依次执行sdio_reg_init.sh和mmc_load.sh

  解绑和绑定mmc bus通用的做法是：

echo 10020000.himci.SD  > /sys/bus/platform/drivers/himci/unbind 2>/dev/null
sleep 2
echo 10020000.himci.SD  > /sys/bus/platform/drivers/himci/bind 2>/dev/null

  解绑和绑定SDIO设备后，himci会对SDIO设备重新探测，即第五步就会打印出对应的commands流

6.5追踪MMC信息：

cat /sys/kernel/debug/tracing/trace_pipe | grep mmc1 &

  其实这一步需要在第4步之前执行，追踪第4步中的信息，不然信息会遗失

  3556上是第5步执行以后，执行cat /proc/mci/mci_info，便检测到SDIO设备

7.加载网卡驱动

insmod rtl8189FS.ko

  驱动会与SDIO设备进行匹配并绑定。若绑定成功，执行ifconfig -a，会显示WLAN0设备：

wlan0     Link encap:Ethernet  HWaddr A0:9F:10:41:8B:D5  
          BROADCAST MULTICAST  MTU:1500  Metric:1
          RX packets:0 errors:0 dropped:0 overruns:0 frame:0
          TX packets:0 errors:0 dropped:0 overruns:0 carrier:0
          collisions:0 txqueuelen:1000 
          RX bytes:0 (0.0 B)  TX bytes:0 (0.0 B)

8.解释第6节第（4）步中sdio_reg_init.sh脚本作用

  SDIO1的管脚分为：SDIO1_CCLK_OUT、SDIO1_CCMD、SDIO1_CDATA0、SDIO1_CDATA1、SDIO1_CDATA2、SDIO1_CDATA3共6个管脚

  参考Hi3556V200_PINOUT_CN.xlsx文档，发现SDIO管脚默认配置为了GPIO管脚。

  因此使用海思的himm工具，修改管脚复用寄存器的值，将以上六个管脚配置为SDIO管脚，而不是GPIO。

9.解释第五节第（4）步中mmc_load.sh在系统启动时的调用逻辑

  Linux系统启动时候，调用顺序/etc/init.d/rcS –> S80mpp –> /app/bootapp –> ./mmc_load.sh
